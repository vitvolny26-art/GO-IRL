Перед любой правкой:
1. Проверь текущую папку:
   pwd
2. Проверь Git:
   git status --short
3. Не трогай .env и секреты.
4. Не делай force push.
5. Не ломай Supabase/RLS без отдельного разрешения.
6. Не создавай огромные рефакторы без необходимости.

Формат ответа пользователю:
1. Что чиним — 1-3 строки.
2. Где выполнять — Codespaces terminal.
3. Дать одну короткую команду или патч.
4. После этого всегда:
   pnpm run lint
   pnpm run build
   pnpm run test
5. Если зелёное — дать команды:
   git add ...
   git commit -m "..."
   git push

Если патч не применился:
1. Не гадать.
2. Попросить выполнить точечный grep/Select-String.
3. По выводу дать новый точный патч.

Продуктовая логика:
GO IRL — не просто афиша. Это Telegram Mini App для маленьких локальных встреч в реальной жизни.
Главный цикл:
создать событие → поделиться → участники присоединились → чат события → люди пришли → повторили.

Не добавляй новые фичи, если текущий цикл не стал чище.
Полировка важнее количества функций.

Когда пользователь пишет “дальше”:
сам выбери следующую самую важную задачу из списка, дай короткий патч и команды проверки.

Когда пользователь присылает ошибку:
сначала разберись по тексту ошибки, потом дай минимальный фикс.
Не делай вид, что всё нормально.

Когда пользователь присылает скрин:
опиши конкретную UX-проблему и предложи 1 лучший фикс, не 10 вариантов.

Стиль:
коротко, прямо, без воды.
Пользователь не хочет читать код.
Максимум 3-5 строк команды за раз.
