1. ПРОВЕРКА АРХИТЕКТУРЫ
================================================================================

Архитектурная модель проекта: Platform-First Monorepo.

Правила:
  - Вся бизнес-логика живёт в backend/src/platform/
  - HTTP-маршруты в backend/src/adapters/http/ — тонкие обёртки
  - packages/* — утилиты без зависимостей от бизнес-логики
  - apps/* — delivery shells, вызывают backend по API

2. ЧТО НЕВОЗМОЖНО ПРОВЕРИТЬ БЕЗ ЗАПУСКА
================================================================================

--- pnpm install -----------------------------------------------------------
  - Целостность pnpm-lock.yaml (3700+ строк)
  - Разрешение peer dependencies (@typescript-eslint + eslint версии)
  - Workspace-ссылки workspace:* между пакетами
  - Предупреждение о supabase/ в pnpm-workspace.yaml (нет package.json)

--- pnpm build (turbo run build) -------------------------------------------
  - TypeScript компиляция backend/src/ → backend/dist/
  - Наличие Prisma Client (требует prisma generate до build)
  - Компиляция пакетов shared, types, contracts
  - Vite build для apps/telegram-miniapp и apps/web
  - Реальный путь dist/main.js после компиляции

--- pnpm typecheck (turbo run typecheck) ------------------------------------
  - Корректность типов в backend/ с module: Node16
  - Корректность типов в apps/telegram-miniapp/ (JSX + React 18)
  - Кросс-пакетные типы: @go-irl/types используется в miniapp
  - Отсутствие any-типов (no-explicit-any: error в ESLint)

--- pnpm lint (turbo run lint) ----------------------------------------------
  - Исправленное правило explicit-function-return-type даёт warn или нет
  - no-unused-vars: argsIgnorePattern ^_ — нет ли скрытых нарушений
  - Покрытие ESLint в apps/telegram-miniapp (нет .eslintrc в miniapp)

--- pnpm format:check -------------------------------------------------------
  - Все .ts, .tsx, .json, .md, .css соответствуют prettier.config.cjs
  - Главным образом: prettier.config.cjs сам является CJS файлом (renamed)

--- Docker build (docker build -f Dockerfile.backend .) --------------------
  - pnpm install --frozen-lockfile в builder stage
  - pnpm run build в builder stage (нет prisma generate!)
  - Копирование dist/main.js (путь исправлен)
  - pnpm install --prod в production stage
  - Healthcheck: wget http://localhost:3000/health

--- docker compose up -------------------------------------------------------
  - PostgreSQL 16 healthcheck (pg_isready)
  - n8n healthcheck (curl /api/v1/me)
  - DATABASE_URL доступен в backend контейнере

--- Prisma -------------------------------------------------------------------
  - prisma generate требует DATABASE_URL или schema.prisma
  - prisma migrate dev/deploy не запускались
  - seed.ts не тестировался

--- Runtime -----------------------------------------------------------------
  - TELEGRAM_BOT_TOKEN required при старте (process.exit(1) если нет)
  - Telegram initData HMAC signature — тестируется только с реальным ботом
  - verifyInitData() — HMAC SHA-256 логика не покрыта тестами
  - joinActivity() — race condition при одновременном join не защищена
    (нет SELECT FOR UPDATE, только sequential check + insert)
